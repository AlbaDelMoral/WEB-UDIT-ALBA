BOOTSTRAP
Runtime
Requirement



What is it?







Personal Reflection
No Node.js




Bootstrap is a free, open-source tool that helps developers quickly build responsive, mobile-friendly websites. It includes ready-made components like buttons, forms, and navigation, plus a grid system and plugins, making it easier to create consistent and attractive designs.



When I explored Bootstrap’s website, what stood out to me was how polished and structured everything felt. It’s clear that Bootstrap focuses on making web design faster and more consistent by providing ready-made components and a strong grid system. I liked how organized their documentation was and how easy it seemed to get started, even for someone not deeply into coding. However, I also noticed that the design style felt a bit generic — many sites built with it can look quite similar unless you customize things heavily. Still, I could see why it’s so popular; it’s reliable, professional, and makes responsive design much simpler.
TAILWIND
Runtime
Requirement



What is it?







Personal Reflection
Requires Node.js




Tailwind CSS is a utility-first CSS framework that helps developers style websites by applying small, reusable classes directly in the HTML. Instead of using pre-designed components, it gives full control over design, making it easy to create custom, responsive layouts quickly and consistently.



AWhen looking into Tailwind, it felt more modern and creative compared to Bootstrap. I liked how it gives designers more control by letting you style things directly through utility classes instead of relying on pre-made themes. The website made it seem very flexible and efficient once you understand how it works. At the same time, it did look a bit overwhelming at first — there are so many classes to learn, and it might take time to get used to. But overall, I found Tailwind really interesting because it encourages more individuality in design while still keeping a clean and consistent structure.

Personal Reflection

Material UI (MUI) is an open-source React component library that implements Google's Material Design system. It provides prebuilt, customizable components that allow developers to build user interfaces for web and mobile applications more quickly and efficiently by following Google's design guidelines, which ensure a consistent and modern user experience.



When looking into Material UI, I was really impressed with the quality and beauty of their components. They showcase high standards of function and form, showing that they take into account aesthetics and functionality, which I liked. However this means that there is not a lot of variety in their designs since they follow Google's Material Design. Overall it has a lot of explanation, demos... in order to learn and implement their components. It is a good site, not too crazy about it.
SHADCN UI
Runtime
Requirement



What is it?






Personal Reflection
Requires Node.js




Shadcn/ui is a set of beautifully-designed, accessible components and a code distribution platform. Works with your favorite frameworks and AI models. Open Source. Open Code. This is not a component library. It is how you build your component library.



After researching Shadcn UI, I found it to be a really interesting and modern approach to building user interfaces. What makes it stand out is that instead of installing a full library, you copy and paste React components directly into your project, which gives you full control over customization. The website looked very clean and well-organized, with examples showing how it combines Tailwind CSS and Radix primitives to create accessible and visually appealing designs. I liked how flexible it seemed and how it encourages developers to understand their code rather than rely on pre-built styles. However, I also noticed that it might be a bit technical for beginners, since it requires some knowledge of React and Tailwind to use effectively.

GSAP, or GreenSock Animation Platform, is a powerful JavaScript animation library that is used by web developers and designers to create interactive and engaging websites. GSAP is known for its versatility, speed, and ease of use, making it a popular choice among developers worldwide.



When I first researched GSAP I was surprised at its modern and vibrant feel. I like their website because cleraly shows what they are about and also in a catchy and attractive way, not like other websites that might feel too technical. When looking into how it works, at first I was a bit overwhelmed because animations seem very difficult to grasp, however even tho the code is a bit harder, the website really helps by explaining it nicely and in the most simple way. One thing I really enjoyed was the scroll animations, since they make the website feel more organic and interactive. I like the tutorials they have and overall seems a good tool for creating smooth animations that can make my website much more interesting.

FLOWBITE
Runtime
Requirement



What is it?





Personal Reflection
Requires Node.js




Flowbite is an open-source library of web components built with the utility-first classes from Tailwind CSS. It also includes interactive elements such as dropdowns, modals, datepickers.



After looking through the Flowbite website, I found it to be a well-designed and useful resource for anyone working with Tailwind CSS. It offers a wide range of pre-built components like navigation bars, modals, and forms, which can save a lot of time when building websites. I liked how the site included interactive examples and clear documentation, making it easy to see how each component works. It seems great for rapid prototyping and creating consistent designs without starting from scratch. However, I also noticed that using Flowbite properly would require some knowledge of Tailwind, and customizing certain parts might be more limiting compared to designing everything manually. Overall, it looks like a very efficient and professional tool for building modern websites quickly.

What is it?





Personal Reflection
SaaS (No Local installation)




Hygraph is a headless content management system (CMS) that delivers content as data via API to any platform instead of being tightly coupled to a specific website, web app, or mobile app.



When I looked into Hygraph, what stood out to me was how modern and flexible it felt compared to traditional content management systems. The idea of a “headless CMS” that uses GraphQL makes it seem very efficient, especially for managing and delivering structured content across different platforms. The website presents it as something powerful for developers, though it also seems a bit intimidating for beginners since it relies heavily on APIs and a more technical setup. Overall, I liked how clean and professional the platform looked, and it gave me the impression of a tool designed for large-scale, well-organized digital projects.

Netlify CMS felt a lot more approachable and beginner-friendly compared to others I researched. My Critical Reflection



Looking back on this project, I’ve learned a lot about how different web tools and frameworks work and how they fit into modern web design. My research helped me see how each program has its own strengths, but also how they often connect through the same systems, like Node.js. I found that Node.js made things easier by helping manage and install different tools, but at first, it was a bit confusing to understand how it all worked together. As I explored more sites and frameworks, I started to appreciate how much goes on behind the scenes to make websites run smoothly. This process taught me to be more patient and curious when learning new technologies, and it gave me a better understanding of the balance between creativity and the technical side of web design.
